## 3. 整体页面框架

平台采用固定左侧导航和右侧内容区域。

```text
┌──────────────┬──────────────────────────────────────┐
│ 达人运营平台  │ 页面标题                 页面主要操作 │
│              ├──────────────────────────────────────┤
│ 首页工作台    │                                      │
│ 达人管理      │                                      │
│ 项目管理      │              当前页面内容             │
│ 内容管理      │                                      │
│              │                                      │
└──────────────┴──────────────────────────────────────┘
```

左侧导航始终保持显示，当前所在菜单使用明显的选中状态。

第一版导航包括：

```text
达人运营平台
├── 首页工作台
├── 达人管理
├── 项目管理
└── 内容管理
```

右侧页面顶部统一包含：

- 当前页面标题
- 简短页面说明
- 当前页面的主要操作按钮
- 面包屑导航，仅在详情页等二级页面显示

## 5. 首页工作台

## 5.1 页面目标

让运营人员进入平台后快速了解数据规模、内容表现和数据同步情况。

## 5.2 页面结构

```text
页面标题：工作台

[达人总数] [项目总数] [内容总数] [累计播放量] [累计互动量]

[媒体达人分布]       [媒体内容分布]

[最近录入内容]

[数据同步异常]
```

## 5.3 核心数据卡片

- 达人总数
- 项目总数
- 内容总数
- 累计播放量
- 累计互动量

互动量计算规则：

```text
互动量 = 点赞数 + 评论数 + 收藏数 + 转发数
```

点击达人、项目或内容数量卡片，可以进入对应列表。

## 5.4 最近录入内容

展示最近录入的内容：

- 内容标题
- 达人
- 媒体平台
- 发布时间
- 同步状态
- 录入时间

点击内容标题进入内容详情页。

## 5.5 数据同步异常

展示最近同步失败或长时间未更新的内容：

- 内容标题
- 媒体平台
- 失败原因
- 最后同步时间
- 操作

支持操作：

- 立即重试
- 查看同步详情
- 手动修正数据

## 6. 达人管理

## 6.1 达人列表

### 页面顶部

- 页面标题：达人管理
- 页面说明：统一管理不同媒体平台的达人账号
- 主要按钮：新增达人

### 搜索和筛选栏

```text
[搜索达人名称] [媒体平台] [内容赛道] [状态] [查询] [重置]
```

交互规则：

- 输入达人名称后，点击“查询”或按回车执行搜索。
- 下拉选择媒体平台、内容赛道或状态。
- 点击“重置”清空所有条件并恢复完整列表。
- 当前筛选条件在返回列表时应尽量保留。

### 表格字段

| 字段 | 说明 |
|---|---|
| 达人名称 | 点击后进入详情页 |
| 媒体平台 | 达人所属媒体 |
| 账号 ID | 媒体账号标识 |
| 内容赛道 | 达人的主要内容方向 |
| 粉丝数 | 当前最新粉丝数 |
| 负责人 | 公司内部对接人员 |
| 关联内容数 | 点击可查看关联内容 |
| 状态 | 正常或停用 |
| 更新时间 | 最后修改时间 |
| 操作 | 查看、编辑、停用或启用 |

### 空状态

列表没有数据时显示：

```text
暂时还没有达人
新增第一位达人，开始建立达人库
[新增达人]
```

搜索无结果时显示：

```text
没有找到符合条件的达人
[重置筛选]
```

## 6.2 新增达人抽屉

点击“新增达人”后，从页面右侧打开抽屉，不离开当前列表。

抽屉建议宽度为页面的 40% 至 50%，最小宽度保证表单可读。

### 表单分组

#### 基本信息

- 达人名称
- 媒体平台
- 账号 ID
- 主页链接
- 内容赛道
- 粉丝数

#### 联系方式

- 微信号
- 手机号
- 邮箱
- 其他联系方式

#### 内部信息

- 负责人
- 备注

### 底部操作

- 取消
- 保存达人

底部操作栏在抽屉滚动时固定显示。

### 重复校验

输入达人名称并选择媒体平台后，系统检查重复：

- 如果组合已存在，禁止保存并提供“查看已有达人”入口。
- 如果账号 ID 或主页链接已存在，提示疑似重复。
- 疑似重复提示不能静默忽略，需要用户明确确认。

### 保存结果

保存成功后：

1. 关闭抽屉。
2. 显示“达人创建成功”提示。
3. 刷新列表。
4. 新增达人显示在列表顶部。

如果从“新增内容”过程中临时创建达人，保存后返回内容抽屉，并自动选中新建达人。

---

## 6.3 编辑达人抽屉

从列表或详情页点击“编辑”，从右侧打开抽屉。

编辑规则：

- 自动带入当前达人信息。
- 修改达人名称或媒体平台时重新执行唯一校验。
- 粉丝数修改后覆盖原值，不保存历史快照。
- 保存成功后刷新当前列表或详情页。

如果用户修改内容后直接关闭抽屉，需要提示：

```text
当前修改尚未保存，确认离开吗？
```

## 8.2 新增内容抽屉

内容只在正式发布后录入。

### 表单分组

#### 内容信息

- 内容标题
- 内容链接
- 发布时间
- 内容类型

#### 业务关联

- 关联达人，必填
- 关联项目，非必填
- 负责人

#### 补充信息

- 备注

第一版新增时不强制要求运营填写播放和互动数据。保存内容后，系统自动进入数据同步流程。

### 选择达人

达人选择框支持输入达人名称搜索。

每条搜索结果同时展示：

```text
达人名称｜媒体平台｜内容赛道｜账号 ID
```

交互规则：

- 只能选择正常状态的达人。
- 没有匹配达人时，提供“新建达人”入口。
- 选择达人后，内容的媒体平台自动继承达人平台。
- 一条内容只能选择一位达人。

### 选择项目

项目选择框支持搜索项目名称或项目编号。

交互规则：

- 项目不是必填项。
- 默认展示进行中的项目。
- 已完成项目可以通过搜索选择。
- 已归档项目不能用于新增内容。
- 没有合适项目时，提供“新建项目”入口。

### 内容链接校验

- 内容链接不能为空。
- 内容链接格式错误时给出提示。
- 内容链接已经存在时禁止保存，并提供“查看已有内容”入口。

### 保存结果

保存成功后：

1. 关闭抽屉。
2. 新内容显示在列表顶部。
3. 系统创建一条待执行的数据同步任务。
4. 内容同步状态显示为“待同步”。
5. 系统根据同步策略自动尝试首次采集。

## 9. 数据回收交互

## 9.1 总体原则

数据回收采用三种渠道：

```text
优先：媒体官方 API
备用：在合法合规前提下采集公开数据
兜底：运营人员手动录入或修正
```

自动采集存在平台登录、反爬、页面变化和指标不可见等不确定性。因此，即使启用自动采集，也必须保留手动修正能力。

第一阶段建议只选择一个媒体平台进行自动采集试点，验证稳定性后再扩展。

## 9.2 同步状态

| 状态 | 含义 |
|---|---|
| 待同步 | 已录入内容，等待系统采集 |
| 同步中 | 系统正在获取数据 |
| 同步成功 | 最近一次同步成功 |
| 同步失败 | 最近一次同步未完成 |
| 暂不支持 | 当前媒体暂时不能自动获取 |

## 9.3 自动同步

内容创建后，系统自动执行首次同步。

后续可按固定时间执行定时同步，具体频率在技术方案阶段确定。

同步成功后：

- 更新五项表现数据。
- 更新数据来源。
- 更新最后同步时间。
- 状态变为“同步成功”。

同步失败后：

- 保留上一次成功的数据。
- 状态变为“同步失败”。
- 记录便于运营理解的失败原因。
- 提供“立即重试”和“手动修正”入口。

## 9.4 立即同步

运营可以从内容列表或详情页点击“立即同步”。

交互反馈：

1. 按钮变为“同步中”并暂时禁止重复点击。
2. 同步成功后刷新数据并显示成功时间。
3. 同步失败后显示原因和处理建议。

## 9.5 手动修正数据

点击“手动修正”后打开弹窗，包含：

- 播放量
- 点赞数
- 评论数
- 收藏数
- 转发数
- 修正说明

保存后：

- 更新当前最新数据。
- 数据来源标记为“手动”。
- 记录数据更新时间。
- 自动同步恢复成功后，可以再次覆盖手动值。

“自动同步是否允许覆盖人工修正值”需要在技术方案阶段进一步确定具体优先级规则。

## 9.6 同步详情弹窗

展示最近一次同步信息：

- 同步开始时间
- 同步完成时间
- 数据来源
- 同步结果
- 失败原因
- 获取到的数据

第一版不要求展示复杂技术日志，只展示运营人员能够理解的信息。

## 10. 通用交互规则

## 10.1 抽屉

- 新增和编辑统一使用右侧抽屉。
- 抽屉打开时保留背后的列表状态。
- 抽屉内表单较长时允许内部滚动。
- 保存和取消按钮固定在底部。
- 存在未保存修改时，关闭需要二次确认。

## 10.2 详情页

- 达人、项目和内容均使用独立详情页。
- 详情页保留返回原列表的入口。
- 返回列表时尽量恢复原搜索条件、筛选条件和页码。

## 10.3 表格

- 表头固定，长列表分页展示。
- 数字统一使用千位分隔符。
- 缺失数据统一显示为 `--`。
- 操作较多时，保留常用操作并将其他操作收进“更多”菜单。

## 10.4 操作反馈

- 成功操作使用短提示。
- 校验失败时在对应字段附近展示原因。
- 系统异常使用业务语言解释，并提供重试方式。
- 停用、归档和作废等重要操作需要二次确认。

## 10.5 加载状态

- 页面首次加载显示骨架屏或明确的加载提示。
- 按钮提交后显示处理中状态，避免重复提交。
- 列表刷新时保留页面整体结构，减少视觉跳动。

