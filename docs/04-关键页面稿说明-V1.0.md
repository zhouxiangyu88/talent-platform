## 2. 本轮覆盖页面

本轮概念稿集中展示四个关键场景：

1. 首页工作台
2. 达人管理列表
3. 内容管理列表
4. 新增内容右侧抽屉

## 3. 已体现的设计原则

- 固定深色左侧导航
- 浅灰页面背景和白色内容卡片
- 蓝色作为主要操作色
- 页面标题与主要操作按钮分区明确
- 首页优先展示核心指标
- 列表顶部提供搜索和筛选
- 主要业务数据使用表格承载
- 内容列表突出数据同步状态
- 新增和编辑使用右侧抽屉
- 整体留白宽松，降低信息压迫感

## 4. 页面说明

### 4.1 首页工作台

页面主要包含：

- 达人总数
- 项目总数
- 内容总数
- 累计播放量
- 累计互动量
- 媒体数据分布
- 最近录入内容
- 数据同步异常

首页用于快速判断整体业务规模和需要处理的异常。

### 4.2 达人管理

页面主要包含：

- 达人名称搜索
- 媒体平台、内容赛道和状态筛选
- 达人数据表格
- 新增达人按钮
- 查看、编辑和停用操作

达人名称作为可点击链接，进入独立详情页。

### 4.3 内容管理

页面主要包含：

- 内容标题或链接搜索
- 达人、媒体、项目和同步状态筛选
- 内容表现数据
- 数据来源
- 同步状态
- 最后同步时间
- 立即同步和手动修正入口

同步状态需要同时通过颜色和文字表达。

### 4.4 新增内容抽屉

抽屉主要分为：

- 内容信息
- 业务关联
- 补充信息

关联达人为必填，合作项目为非必填。保存内容后，系统创建数据同步任务。

## 5. 使用说明

本稿用于确认整体布局、视觉方向和信息层级，不是最终开发截图。

需要注意：

- 图像生成可能导致少量中文文字不准确。
- 最终字段名称和业务规则以 PRD 为准。
- 开发阶段会使用真实前端组件重新实现文字、表格和交互。
- 项目管理和三个独立详情页将在后续页面稿中继续补充。

## 6. 下一步确认重点

请重点确认：

1. 深色左侧导航是否符合预期。
2. 蓝色主色是否合适。
3. 页面留白是否足够宽松。
4. 首页数据卡片和图表比例是否合适。
5. 表格信息量是否过多或过少。
6. 右侧抽屉的录入方式是否直观。
