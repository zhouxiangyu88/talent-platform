## 2. 设计目标

平台主要供内容运营和商务长期使用，视觉设计应满足：

1. 数据清晰，重要信息可以快速识别。
2. 操作简单，新用户不需要学习复杂规则。
3. 页面宽松但不过度浪费空间。
4. 表格适合查看和比较大量达人、项目及内容数据。
5. 数据同步状态和异常情况足够醒目。
6. 后续增加图表和数据看板时仍能保持统一。

---

## 3. 视觉关键词

```text
专业
清晰
可信
克制
数据感
宽松
易操作
```

不采用：

- 大面积渐变
- 过度装饰
- 复杂立体阴影
- 高饱和大面积颜色
- 过于紧凑的表格
- 纯数据大屏式深色界面

---

## 4. 整体布局

### 4.1 页面框架

```text
┌──────────────┬──────────────────────────────────────┐
│ 深色侧边导航  │ 页面标题                 主要操作按钮 │
│              ├──────────────────────────────────────┤
│ 首页工作台    │                                      │
│ 达人管理      │          浅灰页面背景                 │
│ 项目管理      │          白色内容卡片                 │
│ 内容管理      │                                      │
└──────────────┴──────────────────────────────────────┘
```

### 4.2 推荐尺寸

| 项目 | 建议值 |
|---|---|
| 左侧导航宽度 | 224px |
| 页面内容最大宽度 | 不固定，随浏览器宽度展开 |
| 页面左右边距 | 32px |
| 页面上下边距 | 24px 至 32px |
| 卡片内边距 | 24px |
| 表格行高 | 56px |
| 抽屉宽度 | 560px 至 680px |

优先适配 1440px 及以上常见办公电脑屏幕，同时保证 1280px 宽度下可正常使用。

## 5. 色彩规范

## 5.1 主色

主色使用专业、沉稳的蓝色。

| 用途 | 色值 |
|---|---|
| 主操作色 | `#246BFD` |
| 主操作悬停 | `#1557D8` |
| 主操作浅色背景 | `#EAF1FF` |
| 链接文字 | `#246BFD` |
| 选中状态 | `#246BFD` |

主色主要用于：

- 主要按钮
- 当前菜单
- 链接
- 选中控件
- 重点数据
- 图表中的主系列

## 5.2 中性色

| 用途 | 色值 |
|---|---|
| 页面背景 | `#F5F7FA` |
| 卡片背景 | `#FFFFFF` |
| 主要文字 | `#1D2129` |
| 次要文字 | `#4E5969` |
| 辅助文字 | `#86909C` |
| 边框 | `#E5E6EB` |
| 表头背景 | `#F7F8FA` |
| 禁用背景 | `#F2F3F5` |

## 5.3 状态色

| 状态 | 颜色 | 使用场景 |
|---|---|---|
| 成功 | `#00A870` | 保存成功、同步成功、正常 |
| 警告 | `#FF7D00` | 待同步、疑似重复、需关注 |
| 错误 | `#F53F3F` | 同步失败、校验错误、作废 |
| 信息 | `#246BFD` | 同步中、一般提示 |
| 中性 | `#86909C` | 停用、归档、暂不支持 |

状态表达必须同时包含文字，不能只依赖颜色。

## 6. 字体规范

优先使用系统中文字体，保证不同电脑上的可读性和加载速度。

```text
-apple-system
BlinkMacSystemFont
"PingFang SC"
"Microsoft YaHei"
Arial
sans-serif
```

| 层级 | 字号 | 字重 | 使用场景 |
|---|---:|---:|---|
| 页面主标题 | 24px | 600 | 达人管理、内容管理 |
| 区块标题 | 18px | 600 | 基本信息、关联内容 |
| 卡片指标 | 28px | 600 | 达人总数、累计播放量 |
| 正文 | 14px | 400 | 表格、表单、说明 |
| 强调正文 | 14px | 500 | 达人名称、内容标题 |
| 辅助文字 | 13px | 400 | 时间、补充说明 |
| 标签文字 | 12px | 500 | 状态标签 |

数字默认使用千位分隔符，例如 `128,000`。

---

## 7. 间距与圆角

采用 4px 的基础间距体系。

常用间距：

```text
4px、8px、12px、16px、24px、32px、40px
```

| 组件 | 建议间距 |
|---|---|
| 页面标题与内容 | 24px |
| 卡片之间 | 16px |
| 区块之间 | 24px |
| 表单字段之间 | 20px |
| 标签与输入框 | 8px |
| 按钮之间 | 12px |

圆角规范：

| 组件 | 圆角 |
|---|---:|
| 按钮、输入框 | 6px |
| 标签 | 4px 或胶囊形 |
| 卡片、抽屉区块 | 8px |
| 弹窗 | 10px |

整体圆角保持克制，避免过度卡通化。

## 8. 左侧导航

### 8.1 视觉样式

- 使用深蓝黑色背景：`#17233D`
- 平台名称使用白色
- 普通菜单使用浅灰文字
- 当前菜单使用浅蓝背景和白色文字
- 菜单图标保持线性、简洁

### 8.2 菜单结构

```text
达人运营平台

工作台
达人管理
项目管理
内容管理
```

### 8.3 状态

- 默认：透明背景
- 悬停：轻微蓝灰背景
- 选中：蓝色半透明背景，左侧可增加 3px 蓝色标识
- 收起：第一版可以暂不支持

---

## 9. 页面标题区

页面标题区包含：

- 页面标题
- 一行简短说明
- 右侧主要操作按钮

示例：

```text
达人管理
统一管理不同媒体平台的达人账号
                                      [+ 新增达人]
```

详情页增加面包屑：

```text
达人管理 / 小鹿
```

---

## 10. 数据卡片

数据卡片用于工作台和详情页。

内容包括：

- 指标名称
- 核心数值
- 单位
- 可选的补充说明

示例：

```text
累计播放量
1,286,420
已排除作废内容
```

视觉规则：

- 白色背景
- 1px 浅灰边框
- 不使用明显阴影，悬停时可增加轻微阴影
- 核心数字使用 28px
- 重要指标可以使用蓝色图标或小色块

---

## 11. 搜索与筛选区

搜索筛选放在白色卡片顶部，表格之前。

```text
[搜索关键词________________] [媒体平台▼] [状态▼] [查询] [重置]
```

规则：

- 搜索框宽度大于普通下拉框。
- 查询使用主按钮。
- 重置使用文字按钮或次要按钮。
- 筛选项较多时允许换行，保持 16px 间距。
- 已生效的筛选条件需要清晰可见。

---

## 12. 表格规范

表格是第一版最重要的信息载体。

### 12.1 样式

- 白色背景
- 表头使用浅灰背景
- 表头文字使用次要文字色
- 表格行高约 56px
- 行之间使用浅灰分割线
- 鼠标悬停时使用极浅蓝色背景
- 不使用密集的竖向分割线

### 12.2 内容展示

- 达人名称、项目名称、内容标题使用蓝色链接。
- 长内容单行截断，悬停后显示完整信息。
- 缺失数据统一显示 `--`。
- 数值右对齐或保持同列对齐。
- 操作列固定在右侧。
- 常用操作直接显示，低频操作放入“更多”菜单。

### 12.3 分页

表格底部展示：

- 总数据量
- 每页条数
- 当前页码
- 上一页和下一页

---

## 13. 按钮规范

### 13.1 主按钮

用于每个页面最重要的操作：

- 新增达人
- 新增项目
- 新增内容
- 保存
- 立即同步

样式：蓝色背景、白色文字。

### 13.2 次要按钮

用于：

- 取消
- 重置
- 编辑
- 返回

样式：白色背景、灰色边框、深色文字。

### 13.3 危险操作

用于：

- 停用
- 归档
- 作废

默认不使用大面积红色，仅在确认弹窗和最终确认按钮中强调风险。

---

## 14. 标签规范

标签用于表达状态和类别。

示例：

```text
[正常] [停用]
[进行中] [已完成] [已归档]
[待同步] [同步中] [同步成功] [同步失败] [暂不支持]
```

标签采用浅色背景和深色文字，避免大面积高饱和颜色。

---

## 15. 表单与抽屉

### 15.1 抽屉结构

```text
┌──────────────────────────────┐
│ 新增达人                 ×   │
├──────────────────────────────┤
│ 基本信息                     │
│ [字段]                       │
│                              │
│ 联系方式                     │
│ [字段]                       │
├──────────────────────────────┤
│              [取消] [保存]   │
└──────────────────────────────┘
```

### 15.2 表单规则

- 标签放在输入框上方。
- 必填字段使用红色星号标记。
- 两个短字段可以一行排列。
- 长文本、链接和备注独占一行。
- 联系方式、基本信息等使用区块标题分组。
- 校验错误显示在字段下方。
- 帮助说明使用辅助文字色。

### 15.3 抽屉底部

取消和保存按钮固定在抽屉底部，内容滚动时始终可见。

## 17. 数据同步视觉

数据同步是平台的重要差异化能力，需要在内容列表和详情页中清晰表达。

### 17.1 同步成功

- 绿色状态标签
- 显示最后同步时间
- 数据正常展示

### 17.2 同步中

- 蓝色状态标签
- 按钮显示加载状态
- 避免重复触发同步

### 17.3 同步失败

- 红色状态标签
- 在详情中显示业务化失败原因
- 提供“立即重试”和“手动修正”

### 17.4 暂不支持

- 灰色状态标签
- 提示当前媒体暂不支持自动采集
- 保留手动录入入口

## 18. 图表方向

第一版只使用简单、容易理解的图表：

- 横向条形图：各媒体达人数量
- 柱状图：各媒体内容数量
- 折线图：后续展示内容数据变化
- 环形图：内容媒体占比

图表颜色以蓝色为主，使用少量绿色、橙色作为辅助系列。

图表必须同时提供文字、数值或提示信息，不能只依靠图形表达。

---

## 19. 空状态与异常状态

### 19.1 首次使用

```text
暂时还没有达人
新增第一位达人，开始建立达人库
[新增达人]
```

### 19.2 搜索无结果

```text
没有找到符合条件的数据
请调整搜索或筛选条件
[重置筛选]
```

### 19.3 页面加载失败

```text
页面加载失败
请检查网络后重新尝试
[重新加载]
```

空状态使用简单线性插图或图标，不使用过于活泼的插画。

## 20. 响应式原则

第一版以电脑端为主。

当页面宽度不足时：

- 左侧导航可以收窄为图标导航。
- 筛选条件允许换行。
- 表格允许横向滚动。
- 重要列保持在左侧。
- 操作列固定在右侧。

第一版不专门建设手机端，但应保证手机浏览时不会完全无法操作。

---

## 21. 参考原则

参考飞瓜数据类产品时，主要吸收：

- 数据指标优先的页面结构
- 清晰的筛选和查询入口
- 表格承载主要业务数据
- 达人、内容和表现数据之间的关联展示
- 数据状态及分析能力的突出表达

本平台不直接照搬其品牌、布局或具体组件，而是根据公司内部运营流程进行简化：

- 第一版减少复杂榜单和行业分析模块。
- 增加项目、达人和内容之间的业务关联。
- 突出内容数据同步状态。
- 保持页面宽松，降低新用户使用门槛。
