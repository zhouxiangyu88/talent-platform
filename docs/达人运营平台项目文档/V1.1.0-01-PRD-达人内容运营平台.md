| 项目 | 内容 |

| 产品名称 | 达人内容运营平台 |

## 2. 项目背景

目前公司使用在线表格维护达人信息和达人发布的内容，主要存在以下问题：

1. 数据长期积累后容易重复录入达人。
2. 达人信息和内容信息缺乏稳定关联。
3. 在线表格不利于多人长期协作、查询和维护。
4. 不同运营人员的录入方式和字段标准不统一。
5. 内容表现数据分散，不方便汇总和打通。
6. 后续难以接入媒体 API、自动采集和数据看板。

因此，需要建设一个结构化的达人内容运营平台，逐步替代现有在线表格。

---

## 3. 产品目标

### 3.1 建立统一达人库

集中维护达人在不同媒体平台上的账号信息，通过唯一规则减少重复录入。

### 3.2 建立达人与内容的关联

录入已发布内容时必须选择对应达人，使每条内容都有明确归属。

### 3.3 建立合作项目库

独立维护合作项目，支持将多条内容归集到同一个项目中。

### 3.4 建立内容数据基础

手动维护内容表现数据，为后续媒体 API、自动采集、ODS 和数据看板建设打基础。

---

## 4. 用户与使用场景

### 4.1 内容运营

- 查询达人是否已经存在。
- 新增或更新达人信息。
- 录入达人已经发布的内容。
- 将内容关联到合作项目。
- 手动回收和更新内容表现数据。
- 查看某位达人历史发布的内容。
- 按达人、媒体、项目或时间查询内容。

### 4.2 商务

- 查看和维护达人资料。
- 查询达人历史合作内容。
- 建立和查看合作项目。
- 查看项目关联的达人、内容及表现数据。

第一版中，内容运营和商务拥有相同的操作权限。

---

## 5. 业务流程

公司的完整线下合作流程为：

```text
确定合作达人
→ 达人产出内容脚本
→ 脚本审核
→ 达人产出 Demo 视频
→ Demo 审核
→ 正式发布
→ 数据回收
```

第一版平台只承接内容发布后的管理环节：

```text
查找或新增达人
→ 建立合作项目（可选）
→ 内容正式发布
→ 录入内容链接并关联达人
→ 关联合作项目（可选）
→ 手动回收和更新表现数据
```

业务规则：

1. 内容必须在正式发布后才能录入。
2. 脚本和 Demo 暂时不在平台中管理。
3. 每条内容必须填写已发布的内容链接。
4. 内容可以不关联合作项目。

---

## 6. 核心业务对象

第一版包含三个核心业务对象：

1. 达人
2. 已发布内容
3. 合作项目

对象关系如下：

```text
一位达人 → 多条内容
一个项目 → 多条内容
一条内容 → 一位达人
一条内容 → 零个或一个项目
```

### 6.1 达人定义

一条达人记录代表一个达人在一个媒体平台上的账号。

例如：

| 达人名称 | 媒体平台 | 系统记录 |
|---|---|---|
| 小鹿 | 小红书 | 达人记录 A |
| 小鹿 | 抖音 | 达人记录 B |

以上两条属于不同达人记录。

达人唯一规则：

```text
达人名称 + 媒体平台 = 唯一达人记录
```

相同组合不能重复录入。

### 6.2 内容定义

一条内容记录代表达人已经在媒体平台正式发布的一条内容。

内容关联规则：

- 一个达人可以关联多条内容。
- 一条内容只能关联一个达人。
- 一条内容必须关联达人。
- 一个内容链接只能录入一次。
- 内容所属媒体自动继承关联达人的媒体平台。

### 6.3 项目定义

一个项目代表一次合作活动、营销活动或内容合作项目。

项目关联规则：

- 一个项目可以关联多条内容。
- 一条内容最多关联一个项目。
- 内容关联项目不是必填项。
- 第一版暂不管理项目预算、合同和结算。

## 8. 功能需求

## 8.1 首页工作台

首页用于展示整体数据概况。

第一版展示：

- 达人总数
- 项目总数
- 内容总数
- 累计播放量
- 累计互动量
- 各媒体平台达人数量
- 各媒体平台内容数量
- 最近录入的内容
- 最近更新数据的内容

互动量计算规则：

```text
互动量 = 点赞数 + 评论数 + 收藏数 + 转发数
```

第一版首页以数据卡片和简单列表为主，暂不建设复杂图表。

---

## 8.2 达人管理

### 8.2.1 达人列表

列表展示：

- 达人名称
- 媒体平台
- 账号 ID
- 内容赛道
- 粉丝数
- 负责人
- 关联内容数量
- 状态
- 更新时间

支持操作：

- 按达人名称搜索
- 按媒体平台筛选
- 按内容赛道筛选
- 查看达人详情
- 编辑达人
- 新增达人
- 停用达人

第一版不直接永久删除达人，避免关联内容失去归属。

### 8.2.2 达人字段

| 字段 | 是否必填 | 说明 |
|---|---:|---|
| 达人名称 | 是 | 当前媒体平台显示的达人名称 |
| 媒体平台 | 是 | 小红书、抖音、B站、视频号等 |
| 账号 ID | 否 | 媒体平台中的账号标识 |
| 主页链接 | 否 | 达人主页地址 |
| 内容赛道 | 是 | 美食、母婴、数码等 |
| 粉丝数 | 否 | 当前最新粉丝数量 |
| 微信号 | 否 | 达人的微信联系方式 |
| 手机号 | 否 | 达人的手机联系方式 |
| 邮箱 | 否 | 达人的邮箱 |
| 其他联系方式 | 否 | 其他联系渠道 |
| 负责人 | 否 | 公司内部对接人员 |
| 备注 | 否 | 其他补充信息 |
| 状态 | 自动 | 正常或停用 |
| 创建时间 | 自动 | 系统生成 |
| 更新时间 | 自动 | 系统生成 |

### 8.2.3 达人校验规则

1. 达人名称不能为空。
2. 媒体平台不能为空。
3. 内容赛道不能为空。
4. 粉丝数不能小于零。
5. 相同“达人名称 + 媒体平台”禁止重复。
6. 主页链接格式不正确时给出提示。
7. 账号 ID 或主页链接已经存在时，提示疑似重复。
8. 粉丝数只保存最新值，不保留历史快照。

### 8.2.4 达人详情

达人详情页展示：

- 达人基本信息
- 媒体账号信息
- 多种联系方式
- 负责人
- 最新粉丝数
- 状态和备注
- 创建时间和更新时间
- 该达人关联的全部内容
- 内容数量及累计表现数据

支持从达人详情页新增关联内容。

### 8.4.3 内容校验规则

1. 内容只能在正式发布后录入。
2. 必须选择一个达人。
3. 内容标题不能为空。
4. 内容链接不能为空。
5. 内容链接必须唯一。
6. 发布时间不能为空。
7. 播放、点赞、评论、收藏和转发数据不能小于零。
8. 内容所属媒体自动继承关联达人的媒体平台。
9. 内容可以不关联合作项目。
10. 停用达人不能用于新增内容。

### 8.4.4 内容详情

内容详情页展示：

- 内容基本信息
- 关联达人
- 所属媒体平台
- 关联项目
- 内容链接
- 发布时间
- 五项表现数据
- 互动总量
- 互动率
- 数据来源
- 数据更新时间

互动率暂定为：

```text
互动率 =（点赞数 + 评论数 + 收藏数 + 转发数）÷ 播放量 × 100%
```

播放量为零时，不计算互动率，显示为 `--`。

## 9. 数据存储要求

第一版录入的数据存储在业务数据库中，不直接进入 ODS。

```text
运营人员通过前端录入
→ 后端进行业务校验
→ 数据写入业务数据库
```

业务数据库主要保存：

- 达人信息
- 项目信息
- 内容信息
- 达人与内容的关联关系
- 项目与内容的关联关系
- 手动回收的表现数据

系统关闭、刷新或重启后，数据不能丢失。

### 9.1 第一版数据原则

- 粉丝数只保存最新值。
- 内容表现数据只保存当前最新值。
- 第一版暂不保存数据变化历史。
- 第一版暂不建设完整 ODS、DWD、DWS 和 ADS 分层。

### 9.2 未来数据扩展

未来的数据链路可以扩展为：

```text
媒体 API / 自动采集
→ ODS 原始数据层
→ 数据清洗和统一口径
→ 按达人、项目、媒体进行汇总
→ 数据看板
```

第一版数据库需要预留：

- 媒体内容 ID
- 数据来源
- 同步状态
- 最后同步时间
- 外部平台原始数据标识

## 10. 状态与删除规则

### 10.1 达人

- 状态包括“正常”和“停用”。
- 停用后历史内容继续保留。
- 停用达人不出现在新增内容的可选列表中。
- 已关联内容的达人不能永久删除。

### 10.2 项目

- 状态包括“进行中”“已完成”和“已归档”。
- 已关联内容的项目不能永久删除。
- 已归档项目仍可查看历史内容和数据。

### 10.3 内容

- 第一版对错误或无效内容采用“作废”，不直接永久删除。
- 作废内容保留历史记录。
- 数据汇总时默认排除作废内容。

---

## 11. 异常与操作反馈

系统需要提供明确、易懂的操作反馈：

- 保存成功
- 保存失败
- 必填字段未填写
- 达人记录已经存在
- 达人账号或主页链接疑似重复
- 内容链接已经存在
- 项目编号已经存在
- 数据格式错误
- 网络或服务器异常
- 数据已被其他人员修改

错误提示应使用业务语言，不能只显示技术错误代码。

---

## 12. 非功能需求

- 页面优先适配常见电脑屏幕。
- 常用操作步骤尽量少。
- 列表加载速度满足日常运营需求。
- 数据录入规则统一。
- 联系方式属于敏感信息，不应出现在无需展示的页面区域。
- 业务数据库需要定期备份。
- 系统应支持后续部署到正式多人使用环境。
- 系统应支持后续扩展媒体 API、ODS 和数据看板。

---

## 13. 第一版验收标准

1. 能新增、编辑、查询和停用达人。
2. 相同“达人名称 + 媒体平台”不能重复创建。
3. 能搜索和筛选达人。
4. 能查看达人详情及其全部关联内容。
5. 能新增、编辑、查询和归档项目。
6. 能查看项目详情及其全部关联内容。
7. 能新增、编辑、查询和作废已发布内容。
8. 新增内容时必须关联一位达人。
9. 一条内容只能关联一位达人。
10. 内容可以选择不关联项目。
11. 相同内容链接不能重复创建。
12. 能手动维护播放、点赞、评论、收藏和转发数据。
13. 首页能够汇总达人、项目、内容和表现数据。
14. 刷新页面或重启系统后数据不会丢失。
15. 第一版不包含脚本、Demo、审核、权限和定时自动同步功能。
16. 第一版支持在内容库手动点击“同步数据”，先以小红书网页解析作为试点；如果抓取失败，需要记录失败原因，并允许运营继续手动维护数据。
