# 达人内容运营平台需求迭代记录 V1.1.1

更新时间：2026-06-23

## 1. 迭代目标

本次迭代目标是把平台从“基础可用”推进到“更接近日常运营使用”：

- 页面去掉测试阶段说明，减少理解成本。
- 列表筛选能力增强，方便运营快速定位数据。
- 内容和达人链接可以直接跳转到外部平台。
- 列表排序固定为录入顺序，不受编辑影响。
- 内容库支持手动触发小红书数据同步试点。

## 2. 本次新增与调整

### 2.1 页面结构优化

- 删除首页概览、达人库、项目库、内容库顶部测试说明区域。
- 首页概览右上角不再显示新增按钮。
- 达人总数、项目总数、内容总数、总互动量仅在首页概览展示。
- 修复四个模块切换时页面残留或切换不完整的问题。

### 2.2 列表筛选增强

达人库新增筛选：

- 达人分类
- 负责人
- 粉丝数区间

项目库新增筛选：

- 负责人

内容库新增筛选：

- 关联项目
- 播放量区间
- 互动量区间

筛选区域已改为紧凑单行布局，字段过多时在筛选条内部横向滚动。

### 2.3 链接跳转

- 达人库中，达人昵称在存在有效主页链接时可点击跳转。
- 内容库中，内容标题在存在有效内容链接时可点击跳转。
- 无效链接或不完整链接不会变成可点击链接，避免误跳。

### 2.4 列表排序规则

达人库、项目库、内容库统一按照录入时间倒序展示：

- 最后录入的记录展示在最前面。
- 编辑记录不会改变列表位置。

### 2.5 小红书数据同步试点

内容库新增“同步数据”按钮。

当前第一版同步范围：

- 仅支持小红书内容链接试点。
- 仅尝试同步点赞数、评论数、收藏数。
- 不自动更新播放量和转发量。

同步成功时：

- 更新点赞、评论、收藏。
- 同步状态变为“同步成功”。
- 写入同步日志。

同步失败时：

- 不覆盖原有人工数据。
- 同步状态变为“同步失败”。
- 记录失败原因。
- 允许运营继续手动维护数据。

## 3. 当前限制

- 当前小红书同步为无依赖网页解析版本，不使用登录态。
- 如果小红书页面触发验证、风控、内容不公开或页面结构变化，可能无法抓到数据。
- 当前不做定时自动同步。
- 当前不处理验证码、扫码登录、Cookie 管理。
- 当前不支持抖音、B站、微博等平台的自动抓取。

## 4. 验收标准

- 四个模块切换后只展示对应模块。
- 首页概览不展示新增按钮，统计卡片只在首页展示。
- 各列表筛选条件可正常生效。
- 编辑记录后，列表顺序不因更新时间变化。
- 内容标题可打开内容链接，达人昵称可打开达人主页链接。
- 内容库点击“同步数据”后，系统能返回同步成功或失败提示。
- 同步失败时能看到失败原因，且不覆盖原有人工数据。

## 5. 后续建议

下一阶段可以继续做：

- 引入 Playwright，提高小红书动态页面抓取成功率。
- 增加同步详情弹窗，展示最近一次同步时间、失败原因和日志。
- 增加批量同步。
- 增加定时同步任务。
- 扩展 B站、微博等公开页面较容易抓取的平台。
- 后续如能获取官方 API 或第三方数据服务，再替换或补充网页解析方案。
