## 2. 项目背景

当前公司主要使用在线表格维护达人信息和达人发布内容，存在以下问题：

1. 表格长期维护后容易重复录入达人。
2. 达人和内容之间缺少稳定关联。
3. 内容数据、达人数据和项目数据不方便打通。
4. 多个运营人员录入标准不统一。
5. 不利于后续做自动数据回收、数据看板和长期运营沉淀。

因此建设一个轻量级达人内容运营平台，先替代核心表格能力，再逐步向自动抓数、数据看板和权限体系演进。

## 3. 当前版本定位

V1.2.1 是当前可线上使用的内部协作版本。

当前版本解决：

- 公司内部成员可以注册、登录、退出。
- 通过邀请码控制谁可以注册。
- 统一维护达人库、项目库、内容库。
- 内容必须关联达人，项目可选关联。
- 内容可以手动维护播放、点赞、评论、收藏、转发。
- 内容库保留“小红书同步数据”实验入口。
- 平台已部署到 PythonAnywhere，可以通过线上地址访问。

当前版本不解决：

- 不做用户之间的数据隔离。
- 不做角色权限区分。
- 不做找回密码和管理员后台。
- 不保证小红书抓数在 PythonAnywhere 免费版可用。
- 不做正式 ODS、DWD、DWS 数仓分层。

## 4. 用户与权限

### 4.1 使用方

平台主要给公司内部两类人员使用：

- 内容运营
- 商务

### 4.2 当前权限规则

当前版本采用轻量登录方案：

- 用户可以通过邀请码注册。
- 用户使用账号密码登录。
- 未登录不能访问平台首页和业务接口。
- 登录后所有用户看到同一套公司共享数据。
- 暂不区分管理员、运营、商务、只读等角色。

### 4.3 数据可见性

当前版本是公司内部协作型系统：

- A 用户录入的达人，B 用户可以看到。
- A 用户录入的项目，B 用户可以看到。
- A 用户录入的内容，B 用户可以看到。

这是符合当前业务预期的共享数据模式。

## 5. 业务流程

公司完整达人合作流程为：

```text
确定合作达人
→ 达人出内容脚本
→ 脚本审核
→ 达人出 Demo 视频
→ Demo 审核
→ 正式发布
→ 数据回收
```

当前版本平台只承接发布后的管理流程：

```text
查找或新增达人
→ 建立合作项目（可选）
→ 内容正式发布
→ 录入内容链接并关联达人
→ 关联合作项目（可选）
→ 手动回收或尝试同步内容数据
```

业务规则：

1. 内容必须发布后才能录入，因为需要填写内容链接。
2. 脚本和 Demo 暂不在平台里管理。
3. 每条内容必须关联一位达人。
4. 每条内容最多关联一个项目。
5. 项目不是内容录入必填项。

## 6. 核心业务对象

当前版本包含三个核心业务对象：

1. 达人
2. 项目
3. 内容

对象关系：

```text
一位达人 → 多条内容
一个项目 → 多条内容
一条内容 → 一位达人
一条内容 → 零个或一个项目
```

### 6.1 达人

一条达人记录代表一个达人在一个媒体平台上的账号。

达人唯一规则：

```text
达人名称 + 媒体平台 = 唯一达人记录
```

例如：

| 达人名称 | 媒体平台 | 系统记录 |
|---|---|---|
| 小鹿 | 小红书 | 达人记录 A |
| 小鹿 | 抖音 | 达人记录 B |

这两条可以同时存在，因为媒体平台不同。

### 6.2 项目

项目代表一次合作活动、营销活动或内容合作项目。

当前项目规则：

- 项目单独建库。
- 内容可以关联项目，也可以不关联项目。
- 当前不管理预算、合同、报价、结算。

### 6.3 内容

内容代表达人已经正式发布的一条内容。

内容规则：

- 一条内容必须有关联达人。
- 一条内容链接只能录入一次。
- 内容链接会做标准化处理，尽量减少同一内容因不同分享链接导致重复录入。
- 内容所属媒体平台自动继承关联达人的媒体平台。

## 7. 当前功能范围

### 7.1 登录注册

当前已实现：

- 登录页
- 注册页
- 邀请码注册
- 账号密码登录
- 退出登录
- 未登录访问自动跳转登录页
- 未登录访问接口返回未授权
- 密码使用哈希保存，不保存明文密码

当前限制：

- 暂无忘记密码。
- 暂无修改密码页面。
- 暂无管理员重置密码页面。
- 如忘记密码，可重新注册账号，或通过后台数据库重置密码。

- 达人总数

- 内容总数

- 各媒体平台达人数量
- 各媒体平台内容数量
- 最近录入内容

统计卡片只在首页展示，达人库、项目库、内容库不重复展示。

- 新增达人

- 防止同平台同名达人重复录入

## 8. 小红书数据同步试点

当前内容库支持单条内容点击“同步数据”。

当前同步范围：

- 仅试点小红书内容链接。
- 仅尝试同步点赞数、评论数、收藏数。
- 暂不自动同步播放量和转发量。
- 暂不做批量同步。
- 暂不做定时同步。

同步成功时：

- 更新点赞、评论、收藏。
- 记录最后同步时间。
- 同步状态显示为“同步成功”。

同步失败时：

- 不覆盖原有人工录入数据。
- 同步状态显示为“同步失败”。
- 记录失败原因。
- 运营仍可手动维护数据。

当前已知限制：

- PythonAnywhere 免费版对外网访问有限制。
- 小红书网页本身有反爬、登录态、验证码和风控。
- 因此线上点击同步时可能出现 `403 Forbidden` 等失败。
- 当前同步功能属于实验能力，不作为 V1.2.1 核心稳定能力。

## 9. 数据存储

当前版本的数据存储在 PythonAnywhere 上的 SQLite 数据库文件中。

数据文件位置：

```text
/home/2814381620/talent-platform/data/talent_platform.db
```

当前保存的数据包括：

- 用户账号
- 登录会话
- 达人信息
- 项目信息
- 内容信息
- 内容表现数据
- 内容同步日志

当前不是 ODS 表，也不是正式数据仓库。

当前数据链路：

```text
用户在前端录入
→ 后端接口校验
→ 写入 SQLite 业务数据库
→ 页面从数据库读取展示
```

后续如果接入自动抓数和数据看板，可以升级为：

```text
媒体 API / 爬虫
→ ODS 原始数据层
→ 清洗转换
→ 业务数据表
→ 数据看板
```

## 11. 当前验收标准

当前线上稳定版需要满足：

1. 用户可以打开线上地址进入登录页。
2. 用户可以通过邀请码注册账号。
3. 用户可以登录和退出。
4. 未登录不能进入业务页面。
5. 可以新增、编辑、查看达人。
6. 同平台同名达人不能重复录入。
7. 可以新增、编辑、查看项目。
8. 可以新增、编辑、查看内容。
9. 内容必须关联达人。
10. 内容可以不关联项目。
11. 内容链接不能重复录入。
12. 达人昵称和内容标题可在列表中点击跳转外部链接。
13. 编辑记录不会改变列表排序。
14. 首页只展示全局统计。
15. 内容数据可以手动维护。
16. 小红书同步失败时不覆盖人工数据，并展示失败原因。

## 12. 当前风险与限制

| 风险/限制 | 当前处理方式 |
|---|---|
| 忘记密码 | 可重新注册账号，或后台重置 |
