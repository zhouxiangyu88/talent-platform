## 2. 版本定位

V1.3.0 是在 V1.2.1 线上稳定版基础上的下一轮需求规划。

本版本重点不是重做平台，而是把现有平台变得更适合长期运营：

- 达人信息录入更规范。
- 项目状态和项目周期逻辑更清楚。
- 内容录入表单更整齐。
- 同步状态从人工字段改成系统字段。
- 为后续多平台数据抓取和高级看板打基础。

---

## 3. 本版本范围

### 3.1 本版本计划做

- 达人分类改为多选标签。
- 媒体平台增加快手。
- 达人主页链接增加按平台校验。
- 手机号和邮箱增加格式校验。
- 梳理项目状态与开始/结束日期关系。
- 内容类型精简为视频、图文、其他。
- 内容新增/编辑表单统一视觉高度。
- 内容表单移除同步状态下拉框。
- 同步状态改为系统自动字段，只在列表和详情页展示。
- 首页概览升级为更强的运营看板第一版。

### 3.2 本版本先做规划，不强行完整实现

- 多平台自动抓数。
- 定时抓数任务。
- 趋势类数据看板。
- ODS 原始数据层。

这些能力需要更稳定的数据来源和部署环境，建议作为 V1.4.0 及后续版本继续推进。

## 4. 达人库需求

### 4.1 达人分类改为多选标签

当前达人分类为单选，V1.3.0 调整为多选标签。

标签池固定为：

```text
影视娱乐、音乐、生活、人文艺术、摄影、旅游、搞笑趣闻、情感、教育、母婴育儿、财经、游戏动漫、健康、运动、科学科普、科技、互联网、职场管理、美食、时尚、美妆、萌宠、汽车
```

页面规则：

- 新增达人时支持多选。
- 编辑达人时支持修改已选标签。
- 达人列表展示多个标签。
- 达人列表筛选支持按标签筛选。

技术规则：

- 第一阶段可以先用一个字段保存多个标签。
- 后续如果标签体系变复杂，再拆成标准标签表。

### 4.2 媒体平台增加快手

达人媒体平台调整为：

```text
小红书、抖音、B站、视频号、快手、其他
```

保留“其他”，避免未来遇到暂未标准化的平台无法录入。

### 4.3 达人主页链接校验

主页链接不是必填项。

如果填写，需要满足通用规则：

- 必须以 `http://` 或 `https://` 开头。
- 不能包含空格。
- 必须是合法 URL。
- 根据媒体平台校验域名。

平台域名校验建议：

| 媒体平台 | 允许域名 | 说明 |
|---|---|---|
| 小红书 | `xiaohongshu.com`、`xhslink.com` | 允许正式链接和分享短链 |
| 抖音 | `douyin.com`、`v.douyin.com`、`iesdouyin.com` | 允许 PC 主页和移动分享链接 |
| B站 | `bilibili.com`、`space.bilibili.com`、`b23.tv` | B站达人主页常见为 `space.bilibili.com/数字ID` |
| 视频号 | `channels.weixin.qq.com`、`weixin.qq.com` | 视频号链接形态不稳定，先宽松校验 |
| 快手 | `kuaishou.com`、`v.kuaishou.com`、`live.kuaishou.com` | 允许主页、分享短链和直播主页 |
| 其他 | 不限制平台域名 | 只校验 URL 合法性 |

错误提示示例：

```text
当前媒体平台为小红书，请填写小红书相关主页链接。
```

### 4.4 手机号校验

手机号不是必填项。

如果填写，按中国大陆手机号做校验：

- 支持输入 `13800138000`。
- 支持输入 `+86 13800138000`。
- 支持中间有空格或短横线。
- 保存前统一清洗为空格和短横线去除后的号码。
- 核心规则为 11 位手机号，且以 `1` 开头，第二位为 `3-9`。

如果是海外电话、座机、机构电话，建议放入“其他联系方式”。

### 4.5 邮箱校验

邮箱不是必填项。

如果填写，需要满足：

- 包含 `@`。
- `@` 前后都有有效内容。
- 域名部分包含 `.`。
- 不能包含空格。
- 保存前自动去掉前后空格。
- 保存前自动转为小写。

一个邮箱字段只保存一个邮箱。多个邮箱放入“其他联系方式”。

## 5. 项目库需求

### 5.1 项目状态与日期关系

当前项目可以选择状态，也可以填写开始日期和结束日期。V1.3.0 需要明确两者关系。

建议把项目状态分为两类：

| 状态 | 状态来源 | 说明 |
|---|---|---|
| 未开始 | 系统根据日期判断 | 当前日期早于开始日期 |
| 进行中 | 系统根据日期判断 | 当前日期在开始日期和结束日期之间 |
| 已结束 | 系统根据日期判断 | 当前日期晚于结束日期 |
| 已归档 | 人工操作 | 项目结束维护，历史保留，默认不再新增关联内容 |

### 5.2 推荐产品规则

- 如果填写了开始日期和结束日期，系统自动计算项目状态。
- 如果未填写日期，允许人工选择状态。
- 已归档是人工状态，不由日期自动决定。
- 已归档项目仍可查看历史数据。
- 新增内容选择项目时，已归档项目默认不优先展示。

### 5.3 业务解释

```text
已结束 = 项目周期结束，但仍可补数据、复盘、查看
已归档 = 业务上封存，默认不再新增关联内容
```

## 6. 内容库需求

### 6.1 内容表单视觉对齐

问题：

- 关联达人和关联项目在同一排。
- 关联达人因为有必填提示，导致两个下拉框视觉高度不一致。

调整方案：

- 必填用 label 上红色星号表示，例如 `关联达人 *`。
- 输入框下方默认不展示提示。
- 只有用户提交但未填写时，才展示错误提示。
- 表单项统一高度，关联达人和关联项目保持对齐。

### 6.2 发布时间字段

发布时间应为日期选择器，不应为下拉选择。

规则：

- 新增内容时必填。
- 用于记录内容实际发布时间。
- 内容必须发布后才能录入。

### 6.3 内容类型精简

内容类型只保留：

```text
视频、图文、其他
```

### 6.4 同步状态字段调整

同步状态不是运营手动录入字段，而是系统字段。

V1.3.0 调整：

- 新增内容表单移除同步状态下拉框。
- 编辑内容表单移除同步状态下拉框。
- 同步状态只在内容列表和内容详情展示。
- 同步状态由系统根据抓数结果自动更新。

同步状态定义：

| 同步状态 | 设置方 | 说明 |
|---|---|---|
| 未同步 | 系统 | 内容刚录入，还没抓过数据 |
| 同步中 | 系统 | 系统正在抓取数据 |
| 同步成功 | 系统 | 最近一次抓数成功 |
| 同步失败 | 系统 | 最近一次抓数失败 |
| 暂不支持 | 系统 | 当前媒体平台暂不支持自动抓取 |

系统维护字段：

- 同步状态
- 最后同步时间
- 失败原因
- 数据来源
- 标准化链接
- 媒体内容 ID
- 创建时间
- 更新时间

运营填写字段：

- 内容标题
- 关联达人
- 关联项目
- 内容链接
- 发布时间
- 内容类型
- 负责人
- 播放量
- 点赞数
- 评论数
- 收藏数
- 转发数
- 备注

## 7. 多平台抓数规划

### 7.1 目标

长期目标是针对每个媒体平台，尽量自动获取：

- 点赞数
- 评论数
- 收藏数
- 转发数
- 播放量或阅读量

### 7.2 可行性判断

| 平台 | 抓取难度 | 当前建议 |
|---|---|---|
| B站 | 相对较低 | 可优先验证公开数据抓取 |
| 小红书 | 中高 | 继续作为试点，但会受登录和风控影响 |
| 抖音 | 高 | 后续评估，需要更强抓取方案 |
| 快手 | 高 | 后续评估，需要更强抓取方案 |
| 视频号 | 很高 | 暂不作为优先抓取平台 |

### 7.3 自动抓取节奏规划

建议未来节奏：

```text
每天 09:00：同步最近 7 天发布的内容
每天 18:00：再同步一次最近 3 天发布的内容
每周一 10:00：同步最近 30 天内容
超过 30 天的内容：默认不自动同步，只支持手动点击同步
```

页面需要展示：

- 最新数据更新时间。
- 最近同步状态。
- 最近失败原因。
- 数据来源：手动、网页抓取、API。

### 7.4 当前部署限制

PythonAnywhere 免费版对外网访问有限制，小红书等平台可能返回 `403 Forbidden`。

因此，多平台自动抓数不建议完全依赖当前免费部署环境。

后续可选方案：

- 本地辅助同步工具。
- 支持外网访问的付费服务器。
- 第三方数据接口。
- 媒体官方 API。

## 8. 首页概览看板升级

### 8.1 看板目标

首页从简单统计卡片升级为运营看板，帮助内容运营和商务快速判断：

- 当前达人资产规模。
- 内容发布表现。
- 哪些平台效果好。
- 哪些内容和项目值得复盘。
- 哪些内容数据同步异常。

### 8.2 V1.3.0 建议先做

V1.3.0 可以优先做“基于当前最新数据”的看板：

- 核心数据卡片。
- 内容播放 Top10。
- 内容互动 Top10。
- 达人贡献 Top10。
- 项目表现 Top10。
- 各平台内容数量占比。
- 各平台互动量对比。
- 同步成功/失败数量统计。

### 8.3 后续再做

趋势图需要历史快照，目前数据库只保存最新值。

后续如果要做趋势，需要增加数据快照能力，例如：

- 每日内容数据快照。
- 每日达人粉丝数快照。
- 每日项目表现快照。

后续可做：

- 7 天、30 天、90 天发布趋势。
- 播放量趋势。
- 互动量趋势。
- 互动率趋势。
- 标签表现分析。
- 平台 × 标签矩阵分析。

## 9. 验收标准

V1.3.0 开发完成后，需要满足：

1. 新增/编辑达人时，达人分类支持多选标签。
2. 达人列表可以展示多个标签。
3. 媒体平台可选择快手。
4. 达人主页链接能根据媒体平台做基础校验。
5. 手机号填写错误时能给出明确提示。
6. 邮箱填写错误时能给出明确提示。
7. 项目状态和开始/结束日期关系清楚，不产生歧义。
8. 内容新增/编辑表单中，关联达人和关联项目视觉对齐。
9. 内容类型只保留视频、图文、其他。
10. 内容新增/编辑表单不再出现同步状态下拉框。
11. 同步状态由系统维护，并在列表和详情页展示。
12. 首页看板展示至少一组排行榜和一组平台分布。
