# 需求迭代记录：V1.3.0 数据规范、同步状态与看板升级

更新时间：2026-06-24

## 1. 迭代背景

V1.2.1 已完成线上部署、登录注册、达人库、项目库、内容库和基础数据管理。

在试用过程中，发现下一阶段需要重点解决：

- 达人分类不够规范，后续筛选和分析不方便。
- 媒体平台缺少快手。
- 达人主页链接、手机号、邮箱缺少格式校验。
- 项目状态和项目日期之间的关系需要明确。
- 内容表单里部分字段视觉不对齐。
- 同步状态不应该让运营手动选择。
- 首页概览需要升级为更有运营价值的看板。

## 2. 本次新增

### 2.1 达人库

新增：

- 达人分类改为多选标签。
- 媒体平台增加快手。
- 主页链接按媒体平台校验。
- 手机号格式校验。
- 邮箱格式校验。

达人分类标签池：

```text
影视娱乐、音乐、生活、人文艺术、摄影、旅游、搞笑趣闻、情感、教育、母婴育儿、财经、游戏动漫、健康、运动、科学科普、科技、互联网、职场管理、美食、时尚、美妆、萌宠、汽车
```

### 2.2 项目库

新增规则：

- 开始日期和结束日期填写完整时，项目状态优先由系统计算。
- 未开始、进行中、已结束由日期判断。
- 已归档由人工操作，用于表示项目封存。

核心解释：

```text
已结束 = 时间结束，但可以继续补数据、看复盘
已归档 = 项目封存，默认不再新增关联内容
```

### 2.3 内容库

新增/调整：

- 内容类型精简为视频、图文、其他。
- 关联达人和关联项目输入框保持视觉对齐。
- 必填提示改为 label 星号和提交后错误提示。
- 发布时间使用日期选择器。
- 内容新增/编辑表单移除同步状态下拉框。
- 同步状态改为系统字段。

同步状态只在列表和详情展示：

```text
未同步、同步中、同步成功、同步失败、暂不支持
```

### 2.4 首页概览

规划首页看板升级：

- 核心指标卡片。
- 内容播放 Top10。
- 内容互动 Top10。
- 达人贡献 Top10。
- 项目表现 Top10。
- 平台内容数量占比。
- 平台互动量对比。
- 同步成功/失败统计。

## 3. 本次调整

### 3.1 同步状态从人工字段改为系统字段

调整前：

- 内容新增或编辑时，同步状态可能作为下拉选项出现。

调整后：

- 运营不能手动选择同步状态。
- 系统根据抓数过程自动更新同步状态。
- 运营只负责录入内容和手动数据。

### 3.2 自动抓数从功能实现改为分阶段规划

多平台自动抓数不是 V1.3.0 的完整上线目标。

本次先明确：

- 抓哪些字段。
- 状态怎么展示。
- 失败怎么记录。
- 后续抓数节奏如何规划。

完整多平台自动抓数建议后续版本继续推进。

## 4. 技术影响

可能涉及：

- 达人分类字段从单值变为多值。
- 达人列表和筛选逻辑调整。
- 媒体平台枚举增加快手。
- 新增 URL、手机号、邮箱校验函数。
- 项目状态计算逻辑调整。
- 内容表单布局 CSS 调整。
- 内容表单字段调整。
- 首页看板接口可能需要增加排行榜和平台分布数据。

## 5. 数据影响

需要注意：

- 现有单个达人分类数据需要兼容为多标签。
- 旧内容中的同步状态数据继续保留，但不再由用户手动编辑。
- 项目状态计算要兼容没有开始日期或结束日期的旧项目。

## 6. 验收标准

1. 达人分类支持多选，且能正常保存和编辑。
2. 快手可以作为媒体平台选择。
3. 主页链接、手机号、邮箱错误时有明确提示。
4. 项目状态和日期关系展示清楚。
5. 内容表单中的关联达人和关联项目视觉对齐。
6. 内容类型只剩视频、图文、其他。
7. 内容新增/编辑表单不再出现同步状态下拉框。
8. 同步状态只由系统维护。
9. 首页至少新增排行榜和平台分布类看板。

## 7. 暂不做

- 完整多平台稳定抓数。
- 定时任务正式上线。
- 趋势图历史数据。
- 账号权限体系。
- 数据导出和密码修改。
