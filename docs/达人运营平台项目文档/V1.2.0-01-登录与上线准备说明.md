# 登录与上线准备说明 V1.2

更新时间：2026-06-23

## 1. 为什么先做登录

平台上线后会有真实达人、项目和内容数据。如果没有登录，任何拿到地址的人都可以查看和修改数据。

第一版登录目标是轻量可用：

- 有账号密码登录。
- 注册需要邀请码。
- 未登录不能访问平台首页和业务接口。
- 暂时不做复杂角色权限。

## 2. 当前登录方案

当前采用站内账号体系：

- 用户表：`users`
- 登录会话表：`sessions`
- 密码不明文保存，使用 PBKDF2 哈希保存。
- 登录成功后通过 Cookie 保存会话。
- 退出登录后删除会话。

页面：

- `/login.html`：登录页
- `/register.html`：邀请码注册页

接口：

- `POST /api/auth/register`：注册并登录
- `POST /api/auth/login`：登录
- `POST /api/auth/logout`：退出登录
- `GET /api/auth/me`：查询当前登录用户

## 3. 邀请码

本地默认邀请码：

```text
talent2026
```

上线时建议通过环境变量覆盖：

```text
INVITE_CODE=你自己的邀请码
```

不要把正式邀请码写进公开文档或发到公开群里。

## 4. 当前限制

- 暂时没有角色权限，登录用户都可以使用完整平台能力。
- 暂时没有找回密码。
- 暂时没有管理员后台。
- 暂时没有账号停用页面，但数据库已预留 `status` 字段。
- Cookie 当前适合第一版轻量使用，上线后如接入 HTTPS 会更安全。

## 5. 上线前检查

上线前需要确认：

- 设置正式 `INVITE_CODE`。
- 确认数据库文件在云平台持久化目录中，避免重启后数据丢失。
- 确认平台监听云平台提供的 `PORT`。
- 确认未登录访问首页会跳转登录页。
- 确认未登录访问业务 API 会返回 `401`。
- 注册一个正式账号后，再把平台链接发给其他人。

## 6. 后续可迭代

- 增加管理员角色。
- 增加运营、商务只读/可编辑权限。
- 增加用户管理页面。
- 增加重置密码能力。
- 接入企业微信、飞书或邮箱登录。
